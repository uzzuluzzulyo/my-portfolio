== SNS 사이트 DB 구조 정리 ==

사용자 정보 (users 테이블)
- SNS 로그인/프로필 화면에서 필요한 정보들:
  * 사용자 번호: (자동으로 1, 2, 3... 증가)
  * 사용자명: (@ 표시되는 고유 아이디)
  * 이메일: (로그인할 때 사용)
  * 비밀번호: (로그인할 때 사용)
  * 표시 이름: (프로필에 보이는 실제 이름)
  * 소개글: (프로필에 보이는 자기소개)
  * 프로필 사진: (프로필 이미지 경로)
  * 가입일: (언제 가입했는지)

게시물 정보 (posts 테이블)
- SNS 피드 화면에서 필요한 정보들:
  * id: (게시물 고유번호 - 자동으로 1, 2, 3... 증가)
  * user_id: (작성자 ID - users 테이블과 연결)
  * caption: (사진에 쓴 글 내용)
  * created_at: (작성일 - 언제 올렸는지)
  * image_url: (작성 시 선택한 Unsplash 이미지의 URL 링크)
  * likes_count: (좋아요 수 - 현재 좋아요 개수를 숫자로 저장)

게시물 작성 방식:
- 1단계: 내용(캡션) 입력
- 2단계: Unsplash API로 랜덤 이미지 여러 개 보여주기
- 3단계: 마음에 드는 이미지 선택 (새로고침으로 다른 랜덤 이미지들 볼 수 있음)
- 4단계: 선택한 이미지 링크를 posts 테이블의 image_url에 저장

댓글 정보 (comments 테이블)
- 게시물 댓글을 위한 정보들:
  * 댓글 번호: (자동으로 1, 2, 3... 증가)
  * 댓글 내용: (댓글에 쓴 글)
  * 댓글 작성자: (누가 댓글을 썼는지 - users 테이블과 연결)
  * 댓글 달린 게시물: (어떤 게시물에 댓글을 달았는지 - posts 테이블과 연결)
  * 댓글 작성일: (언제 댓글을 썼는지)

SNS 테이블 연결 관계 (단순화 버전)
- 사용자 게시물: 한 사용자가 여러 게시물 작성 가능
  - 게시물 작성 시 Unsplash API로 랜덤 이미지 선택
  - 선택한 이미지 링크를 posts 테이블에 저장
- 게시물 내 좋아요: 각 게시물마다 likes_count 필드로 좋아요 수 관리 (별도 테이블 없음)
- 사용자 댓글: 한 사용자가 여러 게시물에 댓글 가능
- 게시물 댓글: 한 게시물에 여러 사용자가 댓글 가능
